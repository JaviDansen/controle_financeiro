Componentes da tela Fase 1

Ponto de entrada do app. Formulário simples com validação Zod antes de qualquer chamada à API.

Campo e-mail — com validação de formato via Zod no client
Campo senha — input seguro, sem exposição no teclado
Botão "Entrar" — dispara POST /auth/login, feedback de loading
Link "Criar conta" — navega para tela de Registro
Link "Esqueci a senha" — fluxo de recuperação por e-mail
Estado de erro — mensagem clara em credenciais inválidas (401)

Fluxo técnico

Zod valida e-mail + senha antes de qualquer request
POST /auth/login → API valida com bcrypt → retorna JWT
Token salvo no Expo SecureStore (equivalente a httpOnly cookie)
React Query invalida cache → redireciona para Home
Store Zustand armazena dados do usuário em memória

Endpoints da API

POST/auth/loginAutentica, retorna JWT
POST/auth/registerCria novo usuário
POST/auth/logoutInvalida sessão
POST/auth/forgot-passwordEnvia e-mail
POST/auth/reset-passwordRedefine senha

Componentes da tela Fase 1

Dashboard principal. Carrega 4 endpoints em paralelo via React Query com caches diferentes.

Header — saudação dinâmica (Bom dia/tarde/noite), nome, mês atual
Card de saldo — receitas totais, despesas totais e saldo líquido do mês
Gráfico de pizza — distribuição de despesas por categoria
Últimas 5 transações — ícone de categoria, título, valor colorido
Barras de metas — progresso visual de cada meta ativa
FAB (+) — botão flutuante para nova transação, acesso rápido

Endpoints e cache (React Query)

GET/dashboard/summarycache 5min
GET/transactions?limit=5cache 2min
GET/goals?active=truecache 5min
GET/users/mecache 30min

Estados necessários UX

Loading — skeleton loaders no lugar dos cards enquanto carrega
Empty state — quando não há transações ainda (primeiro acesso)
Erro de rede — fallback + botão de retry
Pull-to-refresh — invalida cache e recarrega todos os dados
