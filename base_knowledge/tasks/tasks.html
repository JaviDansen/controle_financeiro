Inicialização do servidor Express com TypeScript, CORS e parsing de JSON. Entry point em apps/api/src/index.ts com rota GET /health para verificação de disponibilidade. Configuração de tsconfig.json e ts-node-dev para hot reload em desenvolvimento.
api 13 mai 2026
Verificação de Conexão ao Banco na Inicialização
Função checkDatabase() executada no startup da API que testa a conexão ao PostgreSQL com SELECT 1. Em caso de sucesso exibe ✔ Banco de dados conectado: host:porta/banco; em caso de falha exibe o erro com host e encerra o processo com exit(1). Roda dentro do callback de app.listen() em apps/api/src/index.ts.

Banco de Dados — Drizzle ORM
Schema Drizzle — Fase 1
Definição completa das tabelas da Fase 1 em packages/db/schema/: users (autenticação), categories (categorias por usuário), cards (cartões de crédito/débito), transactions (receitas e despesas) e goals (metas financeiras). Todas as tabelas usam UUID como PK e timestamps automáticos.
banco 13 mai 2026
Pacote @finapp/db — Conexão e Exportações
Configuração do pacote compartilhado @finapp/db com conexão ao PostgreSQL via pool, drizzle.config.ts apontando para a VPS e exportação centralizada de todos os schemas. É importado pela API e pelo mobile como fonte única de verdade dos tipos.
banco 13 mai 2026
Utilitário de Conexão com Fallback — connection.ts
Novo arquivo packages/db/src/connection.ts com duas funções exportadas: buildConnectionString() prioriza DATABASE_URL e constrói a URL a partir de parâmetros individuais como fallback, usando encodeURIComponent na senha para suportar caracteres especiais como @; getConnectionLabel() usa new URL() para exibir o host de forma segura sem expor a senha. Ambas são exportadas pelo packages/db/src/index.ts e usadas pela API.

Primeira migration gerada pelo Drizzle Kit (migrations/0000_new_unus.sql) e aplicada com sucesso no PostgreSQL da VPS. Cria as cinco tabelas da Fase 1: users, categories, cards, transactions e goals. O drizzle.config.ts foi corrigido para carregar o .env da raiz do monorepo e usar buildConnectionString() com suporte a parâmetros individuais.
